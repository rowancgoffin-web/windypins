Theory

Siting Wind climate from NASA POWER / Global Wind Atlas reanalysis (10–50 km resolution), downscaled to your coordinates with a Jackson–Hunt terrain speed-up correction from sampled elevation. data sources · terrain correction
Shearing Power-law extrapolation to hub height (V ∝ hα); α is a fixed, user-set exponent since no on-site mast exists to derive one directly. wind shear
Losses Jensen/Park wake model, direction-weighted by the site wind rose, plus a DNV/IEC-style multiplicative loss waterfall (blockage, availability, electrical, curtailment, and more). wake model · losses
Results Weibull-integrated power curve with ISA air-density correction; uncertainty expressed as P50–P99 exceedance via root-sum-square of component errors. air density · turbine library

Full derivations, known limitations, and roadmap: Manual.

Data sources. Wind: NASA POWER (MERRA-2, ~50km) / Global Wind Atlas (GWC). Terrain: Open-Elevation + simplified Jackson-Hunt correction. Map tiles: OpenStreetMap / OpenTopoMap / Esri.
Backend connection
Data service: turbine-yield-backend.onrender.com (fixed). Falls back to clearly-labelled synthetic demo data if unreachable. Free-tier hosting sleeps when idle — the first request after a quiet period can take up to a minute while it wakes.

Surface roughness manual

This has a first-order effect on the GWA path — GWC data is parametrized by roughness class, and interpolating at a rougher class legitimately lowers hub-height wind (a 0.03→0.1 change can move gross yield ~15-20%). Pick the class matching the site's actual surroundings. Also used for log-law shear extrapolation in the fallback path. See the manual.

Loss assumptions manual

Categories follow the IEC 61400-15-2 loss framework (numbering is this tool's presentation of that taxonomy). Defaults are DNV-typical onshore values. Applied multiplicatively on top of calculated wake loss. Advanced sub-items are collapsed where a screening user rarely needs them.
